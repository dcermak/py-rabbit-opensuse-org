The callback module¶

This module contains the main entry point: the QueueProcessor class. It provides a method to listen to the event bus and invoke user defined callbacks for specific message types.

class py_rodo.callback.QueueProcessor(*, callbacks: Dict[RoutingKey, Callable[[ObsMessageBusPayloadBase], None]], prefix: str = 'opensuse.obs.', queue_url: str = 'amqps://opensuse:opensuse@rabbit.opensuse.org', logger: Logger | None = None, **kwargs)¶
callbacks: Dict[RoutingKey, Callable[[ObsMessageBusPayloadBase], None]]¶

Dictionary of callback functions that are invoked by listen_forever().

The dictionary allows to specify a callback function for each routing key. The callback function then receives the deserialized payload as an instance of the corresponding child class of ObsMessageBusPayloadBase.

invalid_payload_callback(body: bytes, type_err: TypeError, child_cls: Type[ObsMessageBusPayloadBase], routing_key: RoutingKey) → None¶

Callback that is invoked if the construction of a child class of ObsMessageBusPayloadBase from the message body failed.

This implementation only logs the error.

invalid_routing_key_callback(routing_key: str, body: bytes, val_err: ValueError) → None¶

Callback function that is invoked if a message is received with a routing_key that is not in RoutingKey.

This implementation only logs the error.
